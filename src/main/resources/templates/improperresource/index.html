부적절한 자원 해제

설명: 자원을 적절히 해제하지 않으면, 메모리 누수, 파일 핸들 누수 등의 문제가 발생하여 시스템 성능 저하 및 프로그램 충돌을 초래할 수 있습니다.

대책: 모든 자원을 사용한 후에는 명시적으로 해제하거나 자동 자원 관리를 사용하는 구조를 설계하세요.

샘플 코드

취약 코드:

import java.io.BufferedReader;

import java.io.FileReader;

import java.io.IOException;

public void readFile(String filePath) throws IOException {

BufferedReader reader = new BufferedReader(new FileReader(filePath));

// 취약: 자원을 명시적으로 해제하지 않음

String line;

while ((line = reader.readLine()) != null) {

System.out.println(line);

}

}

설명: 위 코드는 파일 읽기 작업 후 자원을 해제하지 않아 파일 핸들이 누적될 수 있습니다. 이는 운영 체제의 파일 핸들 제한에 도달하게 되어 프로그램 실행이 중단될 수 있습니다.

안전 코드:

import java.io.BufferedReader;

import java.io.FileReader;

import java.io.IOException;

; public void readFileSafely(String filePath) throws IOException {

try (BufferedReader reader = new BufferedReader(new FileReader(filePath))) {

// 안전: try-with-resources를 사용해 자원을 자동 해제

String line;

while ((line = reader.readLine()) != null) {

System.out.println(line);

}

}

}

설명: 이 코드는 `try-with-resources`를 사용하여 파일 읽기 작업이 끝난 후 자원이 자동으로 해제되도록 보장합니다.
