취약한 암호화 알고리즘

돌아가기

설명 :

  • 대칭키 암호화: 단일 키를 사용하여 암호화 및 복호화를 수행하며, 키가 노출되면 보안이 손상됩니다.
  • 공개키 암호화: 비대칭 키를 사용하여 보안을 강화하지만, 짧은 키 길이는 공격에 취약합니다.
  • 해시 알고리즘: 데이터 무결성을 검증하지만, MD5와 같은 약한 알고리즘은 충돌 공격에 취약합니다.

대책 :

  • 대칭키 암호화: AES-256과 같은 강력한 알고리즘을 사용하세요.
  • 공개키 암호화: RSA-2048 이상 또는 ECC를 활용하세요.
  • 해시 알고리즘: SHA-256 이상의 강력한 해시 함수와 비밀번호 해싱 기법(PBKDF2, bcrypt 등)을 사용하세요.
  • ECB모드 취약점 학습
  • ECB 및 GCM 암호화 모드 비교
  • 비밀번호 해쉬 취약점 학습
