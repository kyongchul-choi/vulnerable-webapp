제거되지 않고 남은 디버그 코드

설명: 디버그 코드를 제거하지 않고 배포하면 공격자에게 시스템의 내부 정보를 노출하거나, 보안 취 약점이 발생할 수 있습니다. 디버그 코드는 개발 환경에서만 사용되어야 하며, 운영 환경에 포함되지 않도록 주의해야 합니다.

대책: 코드를 배포하기 전에 디버그 관련 코드를 철저히 검토하고 제거하거나, 로깅 및 디버그 설정이 운영 환경에서는 비활성화되도록 설정하세요.

샘플 코드

취약 코드:

public void processRequest(String request) {

// 취약: 디버그 코드가 운영 환경에서도 실행

System.out.println("Debug: Received request - " + request);

// 요청 처리 로직

// ...

}

설명: 위 코드는 디버그 정보를 콘솔에 출력하여 운영 환경에서 시스템 내부 정보가 노출될 위험이 있습니다.

안전 코드:

import org.slf4j.Logger;

import org.slf4j.LoggerFactory;

public class RequestProcessor {

private static final Logger logger = LoggerFactory.getLogger(RequestProcessor.class);

public void processRequest(String request) {

// 안전: 로깅 레벨에 따라 디버그 정보를 출력

if (logger.isDebugEnabled()) {

logger.debug("Debug: Received request - {}", request);

}

// 요청 처리 로직

// ...

}

}

설명: 이 코드는 로깅 프레임워크를 사용하여 디버그 정보를 로깅하며, 운영 환경에서는 디버그 레벨이 비활성화되도록 설정하여 정보를 노출하지 않습니다.
