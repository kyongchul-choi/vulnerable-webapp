잘못된 세세한 정보의 데이터 공개

설명: 잘못된 데이터 공개는 시스템의 내부 정보(예: 스택 트레이스, 경로, 사용자 정보 등)를 노출하여 공격자가 이를 악용할 가능성을 높이는 취약점입니다.

대책: 에러 메시지, 로그, API 응답에 포함된 데이터를 최소화하고 민감한 정보를 제거하세요.

샘플 코드

취약 코드:

public String getUserData(int userId) {

try {

// 사용자 데이터를 데이터베이스에서 조회

return "User Data: " + getUserDataFromDB(userId);

} catch (Exception e) {

// 취약: 상세한 예외 메시지 공개

return "Error: " + e.getMessage();

}

}

설명: 위 코드는 예외 발생 시 상세한 메시지를 반환하여 공격자가 시스템 내부 정보를 얻을 수 있습니다.

안전 코드:

public String getUserDataSafely(int userId) {

try {

// 사용자 데이터를 데이터베이스에서 조회

return "User Data: " + getUserDataFromDB(userId);

} catch (Exception e) {

// 안전: 일반적인 오류 메시지 반환

return "Error: Unable to fetch user data.";

}

}

설명: 이 코드는 예외 발생 시 일반적인 오류 메시지를 반환하여 시스템 내부 정보가 노출되지 않도록 합니다.
