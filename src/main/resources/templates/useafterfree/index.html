해제된 자원 사용

설명: 해제된 자원을 다시 사용하려고 하면 정의되지 않은 동작이 발생할 수 있으며, 메모리 손상, 시스템 충돌, 또는 악의적인 공격자가 이를 악용할 가능성이 있습니다.

대책: 자원을 해제한 후에는 해당 자원을 명시적으로 null로 설정하거나, 다시 사용하지 못하도록 관리하세요.

샘플 코드

취약 코드:

import java.io.BufferedReader;

import java.io.FileReader;

import java.io.IOException;

public void readFile(String filePath) throws IOException {

BufferedReader reader = new BufferedReader(new FileReader(filePath));

reader.close(); // 자원 해제

// 취약: 해제된 자원을 다시 사용

String line = reader.readLine();

System.out.println(line);

}

설명: 위 코드는 `reader.close()`로 자원을 해제한 후 다시 `reader.readLine()`을 호출하여 정의되지 않은 동작을 초래합니다.

안전 코드:

import java.io.BufferedReader;

import java.io.FileReader;

import java.io.IOException;

; public void readFileSafely(String filePath) throws IOException {

try (BufferedReader reader = new BufferedReader(new FileReader(filePath))) {

String line;

while ((line = reader.readLine()) != null) {

System.out.println(line);

}

} // try-with-resources가 종료되면 자원 자동 해제

}

설명: 이 코드는 `try-with-resources`를 사용하여 자원이 자동으로 해제되며, 해제된 자원이 다시 사용되지 않도록 보장합니다.
