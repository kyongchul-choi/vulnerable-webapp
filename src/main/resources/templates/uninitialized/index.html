초기화되지 않는 변수 사용

설명: 초기화되지 않은 변수를 사용하면 예측할 수 없는 동작이나 런타임 오류가 발생할 수 있습니다. 이는 시스템 안정성을 저하시킬 수 있으며, 잘못된 데이터를 처리할 가능성을 높입니다.

대책: 변수를 선언할 때 반드시 초기화하고, 기본 값을 명시적으로 설정하세요.

샘플 코드

취약 코드:

public void processData() {

int value; // 초기화되지 않음

// 취약: 초기화되지 않은 변수에 접근

System.out.println("Value: " + value);

}

설명: 위 코드는 `value` 변수를 초기화하지 않고 사용하여 컴파일 오류를 발생시킬 가능성이 있습니다.

안전 코드:

public void processDataSafely() {

int value = 0; // 명시적으로 초기화

System.out.println("Value: " + value);

}

설명: 이 코드는 변수를 선언할 때 기본 값을 설정하여 초기화되지 않은 변수 사용으로 인한 문제를 방지합니다.
