메모리 버퍼 오버플로우

설명: 메모리 버퍼 오버플로우는 프로그램이 정해진 버퍼 크기를 초과하여 데이터를 기록하려고 할 때 발생합니다. 이로 인해 인접 메모리가 덮어씌워질 수 있으며, 공격자는 이를 악용해 시스템 명령을 실행하거나 데이터를 훼손할 수 있습니다.

대책: 사용자 입력값의 크기를 제한하고, 버퍼 크기를 초과하지 않도록 검증 코드를 작성해야 합니다.

샘플 코드

취약 코드:

// C 언어 예제

char buffer[10];

strcpy(buffer, userInput); // 취약: 크기 제한 없이 입력값을 복사

안전 코드:

// C 언어 예제

char buffer[10];

strncpy(buffer, userInput, sizeof(buffer) - 1); // 안전: 크기 제한

buffer[sizeof(buffer) - 1] = '\0'; // 널 종료 추가
