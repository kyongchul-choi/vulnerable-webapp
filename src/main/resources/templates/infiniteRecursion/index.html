종료되지 않는 반복 재귀함수

설명: 종료 조건이 없거나 잘못된 조건으로 인해 재귀 함수가 무한히 호출되는 경우, 시스템 리소스를 고갈시키고 스택 오버플로우(StackOverflowError)를 유발할 수 있습니다.

대책: 재귀 호출 시 명확한 종료 조건을 설정하고, 재귀 깊이를 제한하여 무한 반복을 방지하세요.

샘플 코드

취약 코드:

public void infiniteRecursion(int count) {

System.out.println("재귀 호출: " + count);

// 취약: 종료 조건 없음

infiniteRecursion(count + 1);

}

설명: 위 코드는 종료 조건이 없으므로 무한히 자신을 호출하게 됩니다. 이로 인해 스택 오버플로우가 발생하여 프로그램이 중단될 수 있습니다.

안전 코드:

public void safeRecursion(int count, int maxDepth) {

if (count >= maxDepth) {

System.out.println("재귀 호출 종료: " + count);

return;

}

System.out.println("재귀 호출: " + count);

safeRecursion(count + 1, maxDepth);

}

설명: 이 코드는 종료 조건(`count >= maxDepth`)을 추가하여 재귀 호출이 일정 깊이에서 종료되도록 보장합니다. 이를 통해 무한 재귀 호출과 스택 오버플로우를 방지할 수 있습니다.
