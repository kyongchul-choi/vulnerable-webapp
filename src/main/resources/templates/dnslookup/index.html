DNS Lookup에 의존한 보안 결정

설명: 보안 결정을 DNS Lookup 결과에 의존하면, 공격자가 DNS Spoofing을 통해 보안 정책을 우회할 수 있는 취약점이 발생할 수 있습니다.

대책: DNS 기반 확인 후 추가적으로 IP 주소를 직접 비교하거나, 인증된 DNS 레코드(DNSSEC)를 활용하세요.

샘플 코드

취약 코드:

import java.net.InetAddress;

public boolean isTrustedHost(String host) throws Exception {

InetAddress inetAddress = InetAddress.getByName(host);

// 취약: DNS 결과만 확인 return inetAddress.getHostName().equals("trusted.example.com");

}

설명: 위 코드는 DNS 결과만으로 신뢰성을 판단하므로, DNS Spoofing 공격에 취약합니다.

안전 코드:

import java.net.InetAddress;

public boolean isTrustedHost(String host) throws Exception {

InetAddress inetAddress = InetAddress.getByName(host);

String ipAddress = inetAddress.getHostAddress();

// 안전: DNS 결과 확인 후 IP 주소 비교 return "192.0.2.1".equals(ipAddress);

}

설명: 이 코드는 DNS Lookup 결과 외에도 IP 주소를 추가로 비교하여, DNS Spoofing 공격을 방지합니다.
