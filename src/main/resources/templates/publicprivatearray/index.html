Public 메소드에서 반환된 Private 배열

설명: Public 메소드에서 Private 배열을 반환하면 외부에서 배열의 내용을 수정할 수 있는 취약점이 발생합니다. 이는 데이터 무결성을 손상시키고 예기치 않은 동작을 초래할 수 있습니다.

대책: 배열의 복사본을 반환하거나, 배열의 값을 읽을 수 있는 메소드를 제공합니다.

샘플 코드

취약 코드:

public class UserData {

private final String[] userRoles = {"ADMIN", "USER"};

// 취약: Private 배열을 직접 반환

public String[] getUserRoles() {

return userRoles;

}

}

설명: 위 코드는 Private 배열을 외부로 직접 반환하여, 외부에서 배열의 내용을 수정할 수 있습니다.

안전 코드:

public class UserData {

private final String[] userRoles = {"ADMIN", "USER"};

// 안전: 배열의 복사본을 반환

public String[] getUserRoles() {

return userRoles.clone();

}

}

설명: 이 코드는 배열의 복사본을 반환하여, 외부에서 원본 배열을 수정할 수 없도록 보장합니다.
