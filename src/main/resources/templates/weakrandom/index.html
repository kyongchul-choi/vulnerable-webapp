적절하지 않은 난수값 사용

취약점 설명

CWE-338: Use of Insufficiently Random Values

난수값이 예측 가능한 경우 보안상의 문제가 발생할 수 있습니다. 암호화 키, 토큰 생성 등에 적절하지 않은 난수값을 사용하면 공격자가 이를 추측할 가능성이 있습니다.

취약한 코드 예제

1. Predictable Random Values:

import java.util.Random;

public class WeakRandomExample {

public String generateToken() {

// 취약: 예측 가능한 Random 사용

Random random = new Random();

return String.valueOf(random.nextInt(100000));

}

}

2. ThreadLocalRandom 사용:

import java.util.concurrent.ThreadLocalRandom;

public class WeakRandomExample {

public int getRandomNumber() {

// 취약: ThreadLocalRandom도 충분히 안전하지 않음

return ThreadLocalRandom.current().nextInt(0, 1000);

}

}

안전한 구현 방법

1. SecureRandom 사용:

import java.security.SecureRandom;

public class SecureRandomExample {

private static final SecureRandom secureRandom = new SecureRandom();

public String generateSecureToken() {

byte[] randomBytes = new byte[16];

secureRandom.nextBytes(randomBytes);

return Base64.getEncoder().encodeToString(randomBytes);

}

}

보안 대책

1. 적절한 난수 생성기 사용

  • 보안적으로 안전한 난수 생성기 사용 (e.g., SecureRandom)
  • 암호화된 환경에서 난수값 생성

2. 난수값 검증

  • 난수값 사용 시 충분한 엔트로피 보장
  • 난수값을 이용한 작업 결과를 정기적으로 검토
